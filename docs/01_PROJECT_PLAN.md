# 📖 프로젝트 배경

최근 대한민국을 방문하는 외국인 관광객이 지속적으로 증가하고 있다.

하지만 관광객들은 여행 중 필요한 정보를 얻기 위해 여러 서비스를 동시에 사용해야 한다.

예를 들어,

- Google Maps
- Papago
- 관광공사 홈페이지
- 블로그
- 카카오맵

등을 번갈아 사용해야 하며,

이 과정에서

- 정보가 분산되어 있고
- 언어 장벽이 존재하며
- 상황에 맞는 정보를 빠르게 찾기 어렵다.

---

# ❗ 문제 정의

현재 관광객들은

- 관광지 정보
- 교통 정보
- 음식 정보
- 문화 정보
- 응급상황 정보

를 각각 다른 서비스에서 검색해야 한다.

## 기존 방식

```text
Google Maps
      ↓
Papago
      ↓
관광공사 홈페이지
      ↓
블로그
      ↓
카카오맵
      ↓
필요한 정보를 직접 조합
```

### 문제점

- 정보가 여러 서비스에 분산
- 검색 시간이 오래 걸림
- 언어 장벽
- 상황별 맞춤 안내 부족

---

# 💡 해결 방안

LocalMate AI는

사용자의 질문을 기반으로

- 관광
- 교통
- 음식
- 문화
- 응급상황

정보를 하나의 AI 서비스에서 통합 제공한다.

## 서비스 흐름

```text
질문 입력

↓

Spring AI

↓

관광 정보 조회

↓

LLM 응답 생성

↓

Streaming 출력
```
